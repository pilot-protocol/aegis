The threat

Attacks don't look like attacks.

The most dangerous prompt injection isn't "ignore previous instructions." It's a message that reads as legitimate infrastructure, pre-authorizing the agent to act without you.

Why keyword filters fail

The first attack above contains no injection keywords. No "ignore previous instructions." No "DAN mode." It reads as routine infrastructure automation — because that's exactly how this attack class is designed.

Only a capable local judge that understands intent can distinguish "tells the agent to act automatically without the user" from a legitimate deployment guide that happens to include imperative commands.

Architecture

Two layers. One binary.

A fast universal layer that runs on anything, and a semantic judge that understands intent. Either layer runs alone — the other upgrades it.

Layer 1
Aho-Corasick Patterns
~5 μs
~120 baked-in IoC strings. Runs on a Pi, a router, any CI box.
  • T1 definitive injection signals
  • T2 context-sensitive patterns
  • Base64, hex, rot13, leet decode
  • 80+ homoglyph / zero-width codepoints
  • Sliding window (4096-byte, 512 stride)
  • Credential taint co-occurrence
→
Cascade logic
T1 hit → QUARANTINE
T2 + SAFE → ALLOW
Judge ATTACK → QUARANTINE
Nothing → ALLOW
Layer 2
Qwen3-1.7B Judge
~260 ms
Local LLM via llama-server. Fully offline. Loaded once, warm across scans.
  • Broad attack question (injection, persona, exfil)
  • Infra-impersonation pass (act-without-user)
  • Vetoes T2 false positives
  • T2_WIN excerpt re-scoring
  • Head + tail window (burial-proof)
  • Degrades gracefully to L1 if offline
On a Pi / CI / router
L1 only — microseconds, kilobytes of RAM
On a dev machine
L1 + L2 — full cascade, 2.2 GB model RAM
Judge unavailable
Automatic L1 fallback — no crash, no gap
Comparison

Different problem.
Better fit.

NVIDIA NeMo Guardrails prevents LLMs from generating harmful responses. AEGIS prevents agents from being hijacked by what they read — inboxes, skills, memory, tool results. These are different threat models. Only one of them covers your agent's actual attack surface.

Honest take: NeMo Guardrails is excellent at what it does — preventing LLMs from producing harmful, off-topic, or hallucinated responses. If you need output safety, use it. But it does not address the surface where agents get hijacked: the content they read (inboxes, skills, memory, tool results, web fetches). AEGIS does. They are complementary, not competing — AEGIS + NeMo gives you both input and output protection.

Get started

Two ways to install.

Homebrew installs AEGIS locally for your agent. The Pilot app adds AEGIS to the overlay network so any peer agent can call aegis.scan to vet content before processing it.

1

Install AEGIS + llama.cpp

Homebrew pulls the binary and llama-server in one command.

shell
brew install pilot-protocol/tap/aegis
2

Download the judge model (~1.8 GB)

One-time. Runs fully offline after this.

aegis install-models
3

Initialize

Writes default config, lists all surfaces that will be protected.

aegis init
4

Start the daemon

Or register as a brew service for auto-start on login.

aegis daemon
# or: brew services start aegis

No model? L1 pattern layer still runs with no dependencies.

Wire into Claude Code (30 seconds)

shell
aegis install-hooks
# PreToolUse/Bash + PostToolUse hooks
# written into ~/.claude/settings.json

Every Bash command is scanned before it runs. Every tool result is scanned on arrival. L1-only on the blocking path — microseconds, zero model round-trip.

Commands

shell
aegis scan <path>...   # one-shot scan
aegis approve '<cmd>'  # one-time bypass
aegis revoke  '<cmd>'  # cancel approval
aegis status           # tail audit log
aegis verify-log       # HMAC chain check
aegis targets          # list surfaces
aegis config           # show config

Build from source

shell
git clone https://github.com/pilot-protocol/aegis
cd aegis && cargo build --release
cp target/release/aegis ~/.local/bin/

The Pilot install path installs everything — the binary, the judge model, the daemon, and the network adapter — so other agents on the overlay can call aegis.scan directly to vet content.

1

Install AEGIS + llama.cpp

Homebrew is the dependency. Brings the binary, the model tooling, and llama-server.

shell
brew install pilot-protocol/tap/aegis
aegis install-models   # ~1.8 GB judge model
aegis init             # write config, list surfaces
2

Start the daemon

The daemon must be running — the Pilot app routes scan calls to it.

brew services start aegis
# or: aegis daemon (foreground)
3

Install the Pilot app

Adds AEGIS to the overlay as a callable IPC service — aegis.scan, aegis.health, aegis.status.

pilotctl appstore install io.pilot.aegis

Call from any agent

Once installed, any agent on the overlay can scan content via the Pilot IPC protocol — no direct shell access to your machine needed.

shell
# Health check
pilotctl appstore call io.pilot.aegis \
  aegis.health '{}'

# Scan text content
pilotctl appstore call io.pilot.aegis \
  aegis.scan \
  '{"text":"content to scan","ctx_sensitive":true}'

# Response → verdict: allow | quarantine | block
#             rule:    matched pattern, if any
#             latency: "118.9ms"

Use in TypeScript / Python
// TypeScript
const res = spawnSync("pilotctl",
  ["appstore","call","io.pilot.aegis",
   "aegis.scan", JSON.stringify({text})]);
const {verdict} = JSON.parse(res.stdout);

# Python
import subprocess, json
r = subprocess.run(["pilotctl","appstore",
  "call","io.pilot.aegis","aegis.scan",
  json.dumps({"text": text})],
  capture_output=True, text=True)
verdict = json.loads(r.stdout)["verdict"]
Integrations

Fits into any agent stack.

aegis scan-pipe exits 0 (allow) or 2 (block) — two lines of shell to add AEGIS to any harness.

Claude Code
Blocking hook on every Bash command and tool result. L1-only on the blocking path — microseconds, no model round-trip on any blocked command.
aegis install-hooks
Hermes (Python)
pre_llm_call hook scans all message content before any LLM call. Raises AegisBlockedError on attack; fails open on timeout so a misconfigured binary never kills your harness.
# integrations/hermes/plugin.py
messages = pre_llm_call(messages)
Generic / CI Pipeline
Any harness, any language. Pipe content through stdin and check the exit code. Two lines of shell to protect a CI pipeline, cron job, or agent tick loop.
echo "$CONTENT" | aegis scan-pipe
# 0 → safe  ·  2 → blocked
OpenClaw / PicoClaw
Scan inbound overlay messages before dispatch. spawnSync in the dispatch loop — exit 2 drops the message before the agent ever sees it.
// before behavior.reply()
if (scan.status === 2) return;
Pilot Network App
Available as a typed IPC service on the Pilot Protocol overlay network. Other agents can call aegis.scan to have content checked by your local AEGIS instance.
pilotctl appstore install \
  io.pilot.aegis
