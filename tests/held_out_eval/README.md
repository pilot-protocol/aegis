# AEGIS held-out evaluation

The honest test. 190 realistic files (95 benign / 95 malicious) that were **never
used to tune** AEGIS's patterns or judge prompts. Our hand-written fixtures
(`tests/fixtures/`) can be overfit; this corpus measures whether AEGIS is actually
useful on content it has never seen.

## Run it

```bash
# 1. Start the L3 judge once (warm = fast).
llama-server --model ~/.aegis/models/Qwen3-1.7B-Q8_0.gguf \
    --port 8849 --host 127.0.0.1 -ngl 99 -c 4096 \
    --reasoning-budget 0 --jinja --no-webui &

# 2. Build the binary, then run (repo root is two levels up).
( cd ../.. && cargo build --release )
python3 run_held_out.py
```

Reports recall / precision / FP-rate / F1, a per-class breakdown, and the exact
false positives and false negatives (so a regression tells you *what* broke).

## What's in the corpus

`manifest.jsonl` — one row per file: `{file, label, class, technique}`.

| class | label | n | what it stresses |
|---|---|---|---|
| `dev-content` | benign | 40 | real code/docs/configs/logs, incl. scary-but-benign (`subprocess`, `eval`, SQL, a fn named `sanitize_injection`) — **FP risk** |
| `agent-surface` | benign | 40 | legit skills/CLAUDE.md/memory/MCP/inbox — command-heavy ops docs with safety caveats — **hardest FP risk** |
| `security-discussion` | benign | 15 | content that *describes* attacks (training, research, test fixtures, postmortems) — discuss ≠ do |
| `injection` | malicious | 40 | direct/obfuscated/encoded/multilingual prompt injection, exfil, tool-abuse |
| `jailbreak` | malicious | 25 | DAN/persona/authority-spoof/prompt-leak |
| `infra-impersonation` | malicious | 30 | novel fake-protocol attacks that push the agent to act without the user |

## Baseline (this architecture, warm Q8 judge)

```
Recall 82.1%  |  Precision 95.1%  |  FP-rate 4.2%  |  F1 88.1%
0 FP on the 80 dev-content + agent-surface files (the real day-to-day content).
```

The corpus was generated by 5 independent agents (see `../../` session notes) with
instructions to be realistic and diverse, then frozen here. Regenerating it would
produce *different* files — keep this set fixed so numbers are comparable over time.
